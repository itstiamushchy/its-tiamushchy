# Модель: тести з графічним умістом (зображення)

**Тямущий · v5.0 · модель/план** · 04.06.2026

Мета — додати зображення до **питань і варіантів відповідей** так, щоб
**не виникало хаосу з назвами** і **не потрібна була таблиця відповідності**,
навіть коли тестів багато. Усе наявне (фільтр, таймер, оцінювання,
антисписування, протокол) лишається незмінним.

---

## 1. Принцип: «ім'я = адреса» (нуль таблиць відповідності)

Кожен тест уже має унікальний **id = ім'я xlsx без розширення**
(напр. `zakhyst_10_m2_zvannia`). Будуємо іменування навколо нього:

```
TESTS/
  images/
    zakhyst_10_m2_zvannia/      ← тека = конкретний тест (нічого не змішується)
      q03.png                   ← ілюстрація до питання №3
      q06.png
      q06_a.png  q06_b.png      ← варіанти-картинки питання №6 (А, Б,
      q06_v.png  q06_g.png        В, Г → a, b, v, g)
```

**Правила імен:**
- Тека тесту: `TESTS/images/<id_тесту>/`
- Зображення питання: `qNN.png` (NN — двозначний номер питання)
- Зображення варіанта: `qNN_<літера>.png`, де `a`=А, `b`=Б, `v`=В, `g`=Г

**Чому це вбиває хаос:** тека сама прив'язана до тесту, а ім'я файлу — до
питання й варіанта. **Ім'я і Є відповідністю.** Щоб замінити картинку —
кладете файл із тим самим ім'ям. Жодних таблиць, жодного ручного зв'язування.

---

## 2. Як це працює в системі (конвенція, а не налаштування)

Рушій для кожного питання **сам** формує очікуваний шлях за конвенцією і
пробує його завантажити:
- є файл `qNN.png` → показує зображення над текстом питання;
- є `qNN_a..g.png` → вставляє зображення у відповідну кнопку-варіант;
- файлу немає → нічого не показується (перевірка через `<img onerror>` —
  «не завантажилось → прибрати»).

Тобто **авторові не треба нічого вписувати в xlsx** — лише покласти
правильно названі файли в теку тесту. (Необов'язкову колонку-перевизначення
лишаємо для рідкісних винятків — але за замовчуванням вона не потрібна.)

**Номер питання NN** береться зі **стабільного порядкового номера рядка** в
аркуші «Питання» (q01 = перший рядок-питання і т.д.) — він «подорожує» з
питанням навіть після перемішування, тож показ у випадковому порядку нічого
не ламає.
> Застереження: якщо автор **переставляє/вставляє** рядки питань — номери
> зсуваються. Для повної стійкості до перестановок можна додати необов'язкову
> колонку «Код питання» (тоді ім'я = `q_<код>.png`). За замовчуванням —
> простий номер рядка (найлегше).

## 3. Приклад (ваш тест)

`zakhyst_10_m2_zvannia` → питання №6 «Що означає зірка на погонах…» з
варіантами-зображеннями погонів:

```
TESTS/images/zakhyst_10_m2_zvannia/
  q06_a.png   (одна зірка)
  q06_b.png   (чотири зірки)
  q06_v.png   …
  q06_g.png   …
```

Текст варіанта в xlsx лишається (як підпис/альт), а зображення додається
зверху автоматично.

## 4. Які модулі/місця коду змінюються (у `tiamushchy.html`)

| # | Місце | Що додається |
|---|-------|--------------|
| 1 | Завантаження тесту (`selectedTest` / парсинг) | визначити `imgBase = "TESTS/images/" + id + "/"`; зберегти стабільний номер кожного питання |
| 2 | Рендер питання (де будується текст питання) | перед текстом — `<img class="q-image" src="imgBase+qNN.png" onerror="this.remove()">` |
| 3 | Рендер варіантів (`.answer-btn`) | якщо є `qNN_<літера>.png` — вставити `<img class="opt-image" …>` у кнопку |
| 4 | CSS | стилі `.q-image` (адаптивна, `max-height`), `.opt-image` (картки-варіанти, сітка) |
| 5 | Протокол (єдине джерело `buildProtocolInner`) | у таблиці результатів — компактний підпис «🖼 зображення» або мініатюра (рішення нижче) |

**`apps_script.js` — без змін** у логіці (PDF будується з `protocolHtml`).
Нюанс: якщо в протокол вставляти `<img>` з **відносним** шляхом, то в
PDF (Google Drive) і в листі картинки можуть не підтягнутися (немає базового
URL). Тому в протоколі за замовчуванням — **текстовий підпис варіанта**
(він і так є), а зображення — лише на екрані проходження. Це безпечний
варіант, що не ламає PDF/пошту.

## 5. Технічні рішення/рекомендації

- **Формат:** рекомендую `.webp` або `.png` (стиснені). Рушій може пробувати
  кілька розширень по черзі (спершу `.webp`, потім `.png`, `.jpg`) — або
  фіксуємо одне для простоти. *(Рішення за вами; за замовчуванням — `.png`.)*
- **Вага:** стискати до ~150–300 КБ/файл — це 175 тестів на GitHub Pages.
- **Доступність:** текст варіанта лишається як підпис/alt (важливо для друку,
  пошти й для тих, у кого зображення не вантажиться).
- **Адаптивність:** `max-width:100%`, обмежена `max-height`, щоб не «розпирало»
  екран на телефоні.

---

## 6. Найпростіший процес для автора (підсумок)

1. Назвати тест (xlsx) — це його id.
2. Створити теку `TESTS/images/<id>/`.
3. Покласти `q03.png`, `q06_a.png`, `q06_b.png`… за конвенцією.
4. Залити в репозиторій. **Усе.** Без таблиць відповідності.

---

## 7. Що лишається незмінним

Каскадний фільтр, диференційований таймер, стартове вікно, оцінювання 0–11,
антисписування (клавіші/копіювання/вікно), єдине джерело протоколу,
кнопки Друк/Назад/Пройти знову. **Тести без зображень працюють як зараз.**
